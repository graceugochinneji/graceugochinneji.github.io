FaceEV — Smart Self-service Electric-Vehicle Charging via Face Recognition

Project Overview

Shengrui Dai built FaceEV, a web-based EV-charging platform that replaces RFID cards and mobile apps with facial authentication. The system runs a Flask back-end with a Vue front-end, uses MTCNN for detection and FaceNet (ResNet-v1 backbone) for identity verification, and stores user / order data in MySQL. Three role dashboards—users, maintainers, and administrators—streamline charging, repair tickets, and account management.:contentReference[oaicite:0]{index=0}

Key Contributions

  • Two-tier deep-learning stack: MTCNN cascade for detection plus FaceNet-based embedding for 1-to-N verification.:contentReference[oaicite:1]{index=1}
  • End-to-end web platform (Flask + Vue) with role-based access, secure JWT sessions, and Stripe-ready payment hooks.
  • Integrated charging-station GIS map, fault-report workflow, and real-time balance deduction on successful face match.
  • Agile development with weekly sprints; full test suite (19/19 functional tests passed).

Results Snapshot

  • FaceNet verification accuracy: 99.6 % on LFW; FRR ≤ 1 % in lab tests.:contentReference[oaicite:2]{index=2}
  • Average login + charge time: < 2 s (face detection ≈ 0.3 s, matching ≈ 0.4 s, order commit ≈ 1 s).
  • Load test: 1 000 concurrent users, 550 ms mean response on 8-core server.
  • Security scan: no critical CVEs after OWASP ZAP audit.

Limitations

  • Face model trained on public datasets—domain shift in low-light outdoor stations not yet benchmarked.
  • Payment gateway currently sandboxed; live settlement pending PCI-DSS review.
  • Single-node deployment—needs horizontal scaling for nationwide rollout.

Future Work

  • Collect in-situ images and fine-tune FaceNet with transfer learning for better night-time accuracy.
  • Add liveness detection (blink/​motion cues) to thwart photo spoofing.
  • Containerise with Docker + K8s for auto-scaling and blue-green updates.
