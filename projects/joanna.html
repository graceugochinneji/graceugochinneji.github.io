Recognition of Handwritten numerical digits
Using Deep learning Network

Project Overview

Liuwei (“Joanna”) Zheng constructed a VGG-16 convolutional network in TensorFlow / Keras, trained on the classic MNIST dataset. After zero-padding, 32 × 32 resizing and pixel normalisation, the model attains a striking 99.4 % test-set accuracy, positioning it among the top performers on this benchmark. :contentReference[oaicite:0]{index=0}

MNIST Dataset GitHub Repo Download Poster

Key Contributions

  • Adopted stacked 3 × 3 kernels, stride 1 + same-padding to deepen receptive fields without parameter bloat. :contentReference[oaicite:1]{index=1}
  • Inserted Batch Normalisation after every conv layer to speed convergence and curb over-fitting.
  • Achieved a final evaluation of loss = 0.054 · accuracy = 99.2 % on held-out test digits. :contentReference[oaicite:2]{index=2}
  • Packaged a PyQt5 / Tkinter GUI and OpenCV pipeline: users can scribble digits or upload photos; the app segments strokes, predicts each digit, and returns results in < 0.1 s.

Results Snapshot

  • Overall accuracy : 99.4 %
  • Average Precision / Recall / F1 : 0.994 / 0.994 / 0.994
  • ROC–AUC : ≈ 1.00 across all ten classes

Limitations

  • Model confidence drops on very faint or light-coloured strokes. :contentReference[oaicite:3]{index=3}
  • Current GUI occasionally shuffles slice order when processing multi-digit photos.
  • Hand-written digits larger than the padded 32 × 32 window may be partially cropped.

Future Work

  • Add adaptive thresholding & dilation to better capture faint ink.
  • Upgrade to MobileNetV3-Small to enable on-device inference.
  • Integrate sequence-to-sequence decoding for free-form cheque or form recognition.
